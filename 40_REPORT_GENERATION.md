# Shadow Report Generation Specification

**Version:** 1.0
**Last Updated:** January 31, 2026
**Purpose:** Complete specification for PDF report generation and sharing

---

## 1. Overview

Shadow generates comprehensive PDF health reports that users can share with healthcare providers, counselors, or keep for personal records. Reports are:

- **Generated locally** - No server required, all processing on-device
- **Privacy-focused** - User explicitly selects what data to include
- **Professionally formatted** - Clean, medical-style layout
- **Customizable** - Date ranges, categories, and detail levels
- **Accessible** - Screen reader compatible, high contrast options

---

## 2. Report Types

### 2.1 Available Report Types

| Report Type | Purpose | Default Sections |
|-------------|---------|------------------|
| **Comprehensive** | Full health summary for doctor visits | All categories for date range |
| **Condition Focus** | Single condition deep-dive | Condition logs, flare-ups, photos, triggers |
| **Supplement Compliance** | Protocol adherence tracking | Intake logs, compliance %, missed doses |
| **Food & Triggers** | Dietary analysis | Food logs, condition correlations |
| **Sleep Analysis** | Sleep pattern review | Sleep entries, quality trends, duration |
| **Fluids Summary** | Fluid tracking overview | Water intake, bowel/urine, menstruation, BBT |
| **Photo Timeline** | Visual progression | Photos by area, chronological |
| **Custom** | User-selected sections | Any combination |

## 3. Report Sections

### 3.1 Section Inventory

| Section ID | Section Name | Data Source | Charts Included |
|------------|--------------|-------------|-----------------|
| `header` | Report Header | Profile info | None |
| `summary` | Executive Summary | Aggregated stats | Key metrics cards |
| `conditions` | Health Conditions | conditions, condition_logs | Severity trend line |
| `flareups` | Flare-Up Log | flare_ups | Frequency bar chart |
| `supplements` | Supplement Protocol | supplements, intake_logs | Compliance pie chart |
| `food` | Food & Nutrition | food_items, food_logs | Meal frequency |
| `sleep` | Sleep Patterns | sleep_entries | Duration bar, quality trend |
| `fluids_water` | Water Intake | fluids_entries | Daily totals bar chart |
| `fluids_bowel` | Bowel Movements | fluids_entries | Bristol scale distribution |
| `fluids_urine` | Urination Log | fluids_entries | Color distribution |
| `fluids_menstruation` | Menstrual Cycle | fluids_entries | Flow calendar, cycle length |
| `fluids_bbt` | Basal Body Temperature | fluids_entries | Temperature line chart |
| `fluids_other` | Other Fluids | fluids_entries | List format |
| `activities` | Activities | activities, activity_logs | Frequency chart |
| `photos` | Photo Documentation | photo_entries | Thumbnail grid |
| `journal` | Journal Excerpts | journal_entries | Text excerpts |
| `triggers` | Trigger Analysis | Cross-referenced | Correlation table |
| `footer` | Report Footer | Generated metadata | None |

### 3.2 Section Detail Levels

Each section can be generated at different detail levels:

| Level | Description | Use Case |
|-------|-------------|----------|
| **Summary** | Key metrics only, no individual entries | Quick overview |
| **Standard** | Metrics + notable entries | Doctor visits |
| **Detailed** | All entries with full notes | Deep analysis |

### 4.2 Design Specifications

| Element | Specification |
|---------|---------------|
| **Page Size** | US Letter (8.5" x 11") or A4 based on locale |
| **Margins** | 0.75" all sides |
| **Header Height** | 1.25" |
| **Footer Height** | 0.5" |
| **Primary Font** | System sans-serif (SF Pro, Roboto, etc.) |
| **Body Text** | 11pt |
| **Section Headers** | 14pt bold |
| **Subsection Headers** | 12pt semibold |
| **Colors** | Earth tone palette from app theme |
| **Charts** | Rendered as vector graphics (SVG embedded) |
| **Photos** | JPEG, max 300 DPI, 2" max width |

| Element | Color | Hex |
|---------|-------|-----|
| Primary headers | Warm brown | #5D4E37 |

### 4.4 Accessibility

- Minimum contrast ratio 4.5:1 for body text
- Chart data also presented in tabular format
- Alt text for all images and charts
- Logical reading order for screen readers
- High contrast mode available (black/white only)

## 6. Data Aggregation Rules

### 6.1 Summary Statistics

| Metric | Calculation | Display Format |
|--------|-------------|----------------|
| Supplement Compliance | (doses_taken / doses_scheduled) * 100 | XX% |
| Average Severity | SUM(severity) / COUNT(entries) | X.X/10 |
| Sleep Duration | wake_time - bed_time - time_awake | Xh XXm |
| Water Intake Daily Avg | SUM(water_ml) / days_in_range | XX oz or XXX mL |
| Condition Trend | Linear regression slope on severity | Improving/Stable/Worsening |
| Cycle Length | Days between menstruation starts | XX days |

### 6.2 Chart Data Points

| Chart Type | Max Data Points | Aggregation if Exceeded |
|------------|-----------------|-------------------------|
| Line chart | 90 days | Weekly average |
| Bar chart | 31 bars | Weekly totals |
| Pie chart | 8 segments | "Other" category for remainder |
| Calendar heat map | 365 days | Monthly summary |

### 6.3 Notable Entry Selection

For "Standard" detail level, include entries where:
- Severity >= 7/10 (conditions)
- Compliance < 50% for day (supplements)
- Sleep duration < 5 hours or > 10 hours
- Flare-ups occurred
- Photos attached
- User-flagged as important
